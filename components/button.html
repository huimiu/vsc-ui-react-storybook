Button

CSS overrides for the Fluent UI v9 <Button>, <SplitButton>, and <MenuButton> components. Covers five appearance variants, three sizes, icon-only mode, and all interactive states.

Import

import './overrides/vscode-button-overrides.css';

Class Reference

Class Target Description
.vscode-primary Button Blue primary action button (Install, Run, etc.)
.vscode-secondary Button Grey secondary / default button
.vscode-outline Button Outlined border-only button
.vscode-subtle Button Minimal chrome — transparent at rest, tinted on hover
.vscode-transparent Button Fully transparent — icon toolbar style
.vscode-small Modifier 22 px tall (Caption 1 text)
.vscode-compact Modifier 15 px tall (Caption 2 text) — badge or inline use
.vscode-icon-only Modifier Square icon button (no text)

Primary

The main call-to-action. Uses --vscode-button-background for the fill.

Default (28 px)

<Button appearance="primary" className="vscode-primary">Install</Button> <Button appearance="primary" className="vscode-primary" disabled>Install</Button> <Button appearance="primary" className="vscode-primary" icon={<AddIcon />}>New File</Button> <Button appearance="primary" className="vscode-primary vscode-icon-only" icon={<AddIcon />} />

Small (22 px)

<Button className="vscode-primary vscode-small">Run</Button> <Button className="vscode-primary vscode-small" disabled>Run</Button> <Button className="vscode-primary vscode-small vscode-icon-only" icon={<PlayIcon />} />

Compact (15 px)

<Button className="vscode-primary vscode-compact">Badge</Button>

Secondary

Default neutral button. Uses --vscode-button-secondaryBackground.

Default (28 px)

<Button className="vscode-secondary">Cancel</Button> <Button className="vscode-secondary" disabled>Cancel</Button> <Button className="vscode-secondary" icon={<FolderIcon />}>Open Folder</Button> <Button className="vscode-secondary vscode-icon-only" icon={<SettingsIcon />} />

Small (22 px)

<Button className="vscode-secondary vscode-small">Details</Button>

Compact (15 px)

<Button className="vscode-secondary vscode-compact">Tag</Button>

Outline

Border-only with a transparent background. Useful for secondary actions alongside a primary button.

Default (28 px)

<Button appearance="outline" className="vscode-outline">Learn More</Button> <Button appearance="outline" className="vscode-outline" disabled>Learn More</Button>

Small (22 px)

<Button className="vscode-outline vscode-small">View</Button>

Compact (15 px)

<Button className="vscode-outline vscode-compact">Link</Button>

Subtle

Minimal chrome — transparent at rest with a soft tint on hover. Ideal for toolbar-adjacent actions.

Default (28 px)

<Button appearance="subtle" className="vscode-subtle">Edit</Button> <Button appearance="subtle" className="vscode-subtle vscode-icon-only" icon={<CloseIcon />} />

Small (22 px)

<Button className="vscode-subtle vscode-small">Undo</Button>

Compact (15 px)

<Button className="vscode-subtle vscode-compact">×</Button>

Transparent

Fully transparent at rest—hover reveals a toolbar-style tint. Used for icon toolbars.

Default (28 px)

<Button appearance="transparent" className="vscode-transparent">Dismiss</Button> <Button appearance="transparent" className="vscode-transparent vscode-icon-only" icon={<SplitIcon />} />

Small (22 px)

<Button className="vscode-transparent vscode-small vscode-icon-only" icon={<CloseIcon />} />

Compact (15 px)

<Button className="vscode-transparent vscode-compact vscode-icon-only" icon={<CollapseIcon />} />

Icon-Only

Add .vscode-icon-only for a square button that removes text padding. Combine with any appearance. Always provide an aria-label.

Default 28 px
Small 22 px
Compact 15 px
<Button className="vscode-primary vscode-icon-only" icon={<AddIcon />} aria-label="Add" /> <Button className="vscode-secondary vscode-small vscode-icon-only" icon={<SettingsIcon />} aria-label="Settings" /> <Button className="vscode-subtle vscode-compact vscode-icon-only" icon={<CloseIcon />} aria-label="Close" />

Split Button

The Fluent <SplitButton> is a two-part control — a primary action and a menu trigger separated by a divider. Override classes go on the .fui-SplitButton wrapper.

Primary Split

<SplitButton appearance="primary" className="vscode-primary">Run Task</SplitButton>

Secondary Split

<SplitButton className="vscode-secondary">Save As</SplitButton>

Menu Button

A single button with a built-in chevron that opens a menu. Uses the same appearance classes as regular buttons.

<MenuButton appearance="primary" className="vscode-primary">New</MenuButton> <MenuButton className="vscode-secondary">Open Recent</MenuButton> <MenuButton appearance="outline" className="vscode-outline">Filter</MenuButton> <MenuButton appearance="subtle" className="vscode-subtle">Sort</MenuButton>

Interactive States

All five appearances support rest, hover, focus, and disabled states. Focus applies a 1 px --vscode-focusBorder outline with 2 px offset.
