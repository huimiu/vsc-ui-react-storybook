Fluent UI → VS Code Overrides

A collection of CSS-only override stylesheets that restyle Fluent UI v9 components to match the VS Code design language. Each component page includes live previews, CSS class references, and copy-ready code snippets.

Components

Usage

Import the override CSS for the component you need, then add the matching .vscode-* class to your Fluent UI component. The override sheet handles the rest — no JavaScript required.

/* Import the override CSS */ import './overrides/vscode-button-overrides.css'; /* Apply the class to your Fluent Button */ <Button className="vscode-primary">Install</Button> <Button className="vscode-secondary">Cancel</Button> <Button className="vscode-primary vscode-small">Small</Button>

Theming

Override styles read from VS Code's CSS custom properties (--vscode-* tokens) at runtime. When your extension runs inside VS Code, the host provides these tokens automatically, so the overrides adapt to every theme (dark, light, high-contrast) without extra work.

Use the theme toggle in the top-right corner of this site to preview both dark and light appearances.

Packaging

The override CSS files in /overrides are standalone — no build step or bundler required. They can be published to npm and consumed via a simple CSS import. Keep them separate from your component logic so they remain tree-shakeable and easy to version.
